# Browser task workflows

## 1. Simple lookup

Use when the user wants information from a webpage.

Steps:
1. Find or open the target URL.
2. Read the key content.
3. Extract only what answers the user's request.
4. Cite the source URL when useful.

## 2. Multi-page lookup

Use when the user wants comparison or summary across pages.

Steps:
1. Identify 2-5 relevant pages.
2. Read each page lightly.
3. Normalize the findings into bullets.
4. Highlight differences.

## 3. Interactive website task

Use when static reading is not enough.

Steps:
1. Open the page.
2. Inspect visible elements.
3. Click or fill one step at a time.
4. Re-check state after each step.
5. Stop before irreversible actions unless the user clearly asked.

## 4. Screenshot workflow

Use when the user wants proof or a visual reference.

Steps:
1. Reach the target state.
2. Capture screenshot.
3. Label what the screenshot shows.
4. If needed, pair it with a short summary.
